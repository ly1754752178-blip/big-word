# 综漫日本生活模拟器 — 蔚蓝档案 × 日常 Galgame 风格改造设计文档

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `F:\AI\no2` 的现有前端全面改造为以「现实日本 + 动漫角色」为核心的生活模拟游戏界面，采用蔚蓝档案的清新视觉与日常系 Galgame 的柔和氛围，仅制作前端原型，无后端。

**Architecture:** 基于现有 Vite + React + TypeScript + Tailwind CSS 技术栈，通过扩展 `GameContext` 状态、新增页面/覆盖层/组件、重写视觉系统来实现改造。所有数据继续使用 mock 数据或本地状态，LLM 交互通过前端 Hook 占位/mock 实现。

**Tech Stack:** Vite 5 + React 18 + TypeScript 5 + Tailwind CSS 3 + Framer Motion + Lucide React。

---

## 全局约束

- **仅前端原型**：不制作后端，API 调用用 mock/占位实现。
- **全中文界面**：除 logo 与视觉元素外，所有文字为中文。
- **禁用 emoji**：所有图标使用 Lucide React。
- **现实日本背景**：不引入非角色幻想/科幻设定。
- **高级 UI/UX**：必须细节丰富、动画灵动、拒绝简陋 MVP。
- **语义化 HTML + 唯一 ID**：便于浏览器测试与可访问性。
- **性能优化**：动画仅 transform/opacity，懒加载非关键资源。

## 1. 世界观与核心体验

### 1.1 世界观：「综漫日本」

这就是**现实中的现代日本**。城市、街道、电车、便利店、大学、公司、节日与现代社会规则全部与现实无异。

唯一的特殊之处在于：来自不同日本动漫、轻小说作品的角色，以普通日本人身份生活在这个社会中。他们是你的同学、同事、邻居、店员、路人。游戏不解释他们为何存在，正如现实不解释任何人为何存在。

### 1.2 玩家身份

玩家扮演一名生活在现代日本的普通人，可选起点：

- **大学生**：平衡学业、打工、社团、求职。
- **社会人**：通勤、职场、租房、周末生活。
- **自由创作者**：小说家、画师、独立游戏开发者、视频创作者。
- **转学生/归国子女**：重新适应日本生活。

### 1.3 核心体验循环

```
早晨起床 → 查看日程/天气 → 出门移动 → 学校/公司/打工
    → 与角色互动/触发事件 → 推进个人目标 → 夜晚回家
    → 结算一天（健康、财务、关系变化） → 进入下一天
```

核心目标不是拯救世界，也不是必须恋爱，而是：

> **像一名真实的现代日本人一样生活、工作、成长、与他人建立关系。**

### 1.4 可玩方向

- **纯生活流**：上课、打工、做饭、通勤、独处，不攻略任何角色。
- **社交流**：与角色建立友谊、恋人、竞争对手等关系。
- **事业流**：提升技能、完成项目、升职加薪或创作作品。
- **探索流**：在城市地图中探索不同地点，触发随机事件。

## 3. 页面与模态框设计

### 3.1 主界面（Home）

**功能**：游戏的核心仪表盘，展示当前时间、地点、天气、角色状态、叙事流。

**内容要求**：
- 顶部：日期、时间、天气球、节日/节气提示、BGM 播放器。
- 左栏：导航 + 当前选中模块的预览面板。
- 中栏：
  - 地点标题与氛围描述。
  - 叙事流（旁白、对话、选项）。
  - 输入框（玩家行动/对话输入）。
- 右栏：
  - 场景印象图（当前地点插画/照片）。
  - 迷你 GPS 地图（当前位置与附近角色）。
  - 周边动态（附近事件、角色动态、通知）。
  - 手机入口。

**改造重点**：
- 视觉全面转向 Blue Archive 清新卡片 + Galgame 奶油色叙事区。
- 叙事区增加选项按钮、对话头像放大、柔和阴影。
- 天气球增加外框与半圆突出效果。

### 3.2 角色图鉴（Character Gallery）

**功能**：浏览所有登场角色，查看资料、好感度、关系状态、专属剧情入口。

**内容要求**：
- 角色卡片网格：头像、姓名、身份标签、当前好感度/关系阶段。
- 筛选：按关系（同学/同事/邻居/恋人等）、按登场作品（原作标签）、按地点。
- 点击卡片进入角色详情页。
- 每个角色必须有完整资料：姓名、出处作品、年龄、职业/身份、性格标签、喜好、与你的关系阶段、最近互动记录。

**改造重点**：
- 卡片采用圆角 + 柔和阴影 + hover 上浮。
- 好感度用爱心/温度条可视化。
- 不使用 emoji，关系阶段用图标+文字（如「陌生人」「同学」「朋友」「密友」「恋人」）。

### 3.3 角色详情（Character Detail）

**功能**：单个角色的完整档案与互动入口。

**内容要求**：
- 顶部：大头像、姓名、出处作品、身份标签、关系阶段。
- 基本信息：年龄、生日、血型、职业、住址、爱好、厌恶。
- 状态：当前心情、位置、正在做什么。
- 互动按钮：聊天、送礼、邀请、查看回忆。
- 事件时间线：与该角色相关的重要事件。
- 关系成长条：从陌生到恋人的多阶段可视化。

### 3.4 关系网络（Relationship Network）

**功能**：可视化玩家与角色之间、角色与角色之间的关系网。

**内容要求**：
- 中心节点为玩家。
- 角色节点按圈层分布（同学圈、职场圈、邻里圈、兴趣圈）。
- 连线表示关系强度与类型（友谊、恋情、竞争、家人）。
- 支持拖拽、缩放、筛选。
- 点击节点查看角色详情。

**改造重点**：
- 使用柔和配色区分关系类型。
- 节点头像放大，hover 显示关系摘要。
- 背景使用浅色调渐变。

### 3.5 日程与日历（Calendar）

**功能**：查看日期、安排日程、触发季节事件。

**内容要求**：
- 月视图网格，显示日本节假日、角色生日、季节活动。
- 今日日程列表：课程、打工、约定、截止日期。
- 事件详情弹窗。
- 支持添加/编辑/删除日程（前端状态即可）。

### 3.6 城市地图（City Map）

**功能**：探索城市地点，查看当前位置与附近角色/事件。

**内容要求**：
- 地图标记：家、学校、公司、便利店、咖啡馆、公园、车站等。
- 地点卡片：名称、类型、当前在场角色、可触发事件。
- 移动操作：点击地点进行移动，消耗时间与体力。
- 迷你地图始终显示在右栏。

### 3.7 生活管理（Life Management）

**功能**：管理玩家的健康、财务、技能、工作/学业。

**子模块**：
- **身体状态**：体力、精神、健康、疲劳、心情、异常状态。
- **财务资产**：现金、银行卡、支出记录、收入记录、固定资产。
- **技能树**：料理、打扫、编程、沟通、创作等日常/职业技能。
- **工作/学业**：课程表、项目进度、绩效、考试、升职路径。

### 3.8 创作工坊（Creative Workshop）

**功能**：管理个人创作项目（小说、漫画、游戏、视频、音乐）。

**内容要求**：
- 项目列表：名称、类型、进度、截止日期、参与角色。
- 项目详情：阶段划分、任务清单、灵感值、完成奖励。
- 角色协作：邀请角色参与项目，影响效率与事件。
- 作品发布后的反响模拟。

### 3.9 商店与换装（Shop & Wardrobe）

**功能**：购买礼物、服饰、家具，影响好感度与个人属性。

**内容要求**：
- 商店分类：便利店、书店、服装店、家电、杂货。
- 商品卡片：名称、价格、效果、库存。
- 购物车与购买确认。
- 已购物品进入背包/衣橱。

### 3.10 回忆相册（Memories）

**功能**：收集与回放重要事件、CG、成就。

**内容要求**：
- 事件时间线：按日期排列的重要回忆。
- CG 画廊：关键剧情截图/插画占位。
- 成就列表：生活向成就（如「第一次独居做饭」「连续通勤 30 天」「与某人成为挚友」）。

### 3.11 手机系统（Phone）

**功能**：游戏内的智能手机，整合通讯与社交。

**应用清单**：
- **短信**：与角色的聊天记录。
- **SNS 动态**：角色发布的日常动态。
- **通知**：日程提醒、事件通知、系统提示。
- **设置**：主题、字体、API、数据管理。
- **地图**：快速查看附近。
- **钱包**：查看余额与消费记录。

**视觉要求**：
- 黑色手机边框 + 浅色屏幕。
- 顶部状态栏：时间、WiFi、电池。
- 应用网格图标统一 Lucide。

### 3.12 AI 叙事剧场（AI Narrative）

**功能**：核心 LLM 交互区域，生成日常剧情与分支选项。

**内容要求**：
- 玩家输入行动或对话。
- LLM 返回旁白、角色对话、选项。
- 支持分支选择、回溯到某节点、重新生成。
- 显示「正在输入」状态。

**视觉要求**：
- 叙事文本像 Galgame 一样分段呈现。
- 选项按钮采用卡片式，hover 有柔和反馈。
- 对话头像圆角矩形，尺寸充足。

### 3.13 系统设置（Settings）

**功能**：游戏内设置与数据管理。

**内容要求**：
- 外观：主题色、字体大小、动画开关、暗色模式。
- LLM：API 配置（占位）、模型选择、温度、提示词模板。
- 数据：导入/导出存档、重置数据。
- 关于：版本信息、制作人员。

## 5. 交互与动画规范

### 5.1 通用交互

- 所有可点击元素有 hover/active/focus 状态。
- 图标按钮 `aria-label` 必填。
- 按钮按压 scale 0.97，150ms ease-out。
- 卡片 hover 上浮 2-4px + 阴影加深。

### 5.2 页面切换

- 模态框进入：scale 0.96 → 1 + opacity 0 → 1，spring 曲线，250ms。
- 模态框退出：反向，180ms。
- 标签页切换：内容淡入淡出，指示器滑动。

### 5.3 列表与网格

- 列表项进入 stagger 30-50ms。
- 网格卡片 hover 时相邻卡片轻微推斥（可选）。

### 5.4 叙事区

- 新消息从下方淡入，200ms。
- 选项按钮依次出现，stagger 50ms。
- LLM 生成时显示柔和脉冲的「输入中」指示器。

## 6. LLM 集成设计

### 6.1 前端占位方案

由于无后端，LLM 调用在前端通过 Hook 占位实现：

- `useLLM()` Hook：接收 prompt 与上下文，返回 mock 流式文本或延迟后的固定回复。
- 所有 LLM 相关状态（正在输入、流式输出、错误）在 `GameContext` 中管理。
- API 配置 UI 可填写，但不真实发送请求（或仅在用户提供 key 时发送，需说明风险）。

### 6.2 提示词组装

- 使用本地 `promptAssembler` 函数（mock）。
- 输入包括：当前时间、地点、玩家状态、在场角色、关系值、历史剧情。
- 输出解析为：旁白、角色对话、选项列表、状态变化。

### 6.3 叙事状态

- 叙事消息类型扩展为：`scene`（旁白）、`dialogue`（对话）、`option`（选项）、`system`（系统提示）。
- 支持分支树：每个选项指向新的剧情节点。
- 支持回溯：删除某条消息后的所有内容，重新生成。

## 7. 性能与可访问性

### 7.1 性能

- 图片使用 WebP/AVIF 占位，声明尺寸避免 CLS。
- 非首屏组件 lazy import。
- 长列表虚拟化（角色图鉴、叙事历史）。
- 动画仅 transform/opacity，避免 layout thrashing。
- 字体使用 `font-display: swap`。

### 7.2 可访问性

- 语义化 HTML：`header`、`main`、`aside`、`nav`、`article`、`dialog`。
- 所有交互元素唯一 `id`。
- 颜色对比度 ≥ 4.5:1。
- 支持键盘导航与 focus visible。
- 尊重 `prefers-reduced-motion`。
- 错误提示使用游戏内通知，不使用浏览器 `alert`。
