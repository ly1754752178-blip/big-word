# BGM 播放器升级设计

> 日期：2026-07-14 | 分支：feature/talent-detail-modal

## 概述

将 BGM 播放器从硬编码单曲升级为动态加载 `public/BGM/` 目录下所有音频文件，支持情景词分类浏览与选歌。**播放器主体 UI 保持不变**，仅增强列表功能。

## 文件规范

### 目录结构

```
public/BGM/
├── order.txt              ← 情景词分类顺序（用户维护）
├── 日常_悠闲时光.mp3
├── 日常_悠闲时光.jpg       ← 同名封面，支持 jpg/png
├── 午后_暖阳.mp3
├── 午后_暖阳.png
├── 夜空_星辰大海.flac
├── 夜空_星辰大海.jpg
└── 夏天_蝉鸣与风铃.mp3
```

### 命名规则

`<情景词>_<曲名>.<扩展名>`

- 情景词与曲名之间用下划线 `_` 分隔
- 曲名中可包含下划线（如 `蝉鸣与风铃`），取第一个下划线之后的全部内容作为曲名
- 封面通过同名匹配：`xxx.jpg` 或 `xxx.png`，优先 `jpg`
- 支持音频格式：`.mp3` `.ogg` `.wav` `.flac` `.m4a`

### order.txt 格式

```txt
1、日常
2、午后
3、夜空
4、夏天
```

- 每行一条：`序号、情景词`
- 序号决定标签栏排列顺序
- 文件中未出现的分类排在最后（按字母序）
- 无此文件时，所有情景词按拼音首字母排序

## 架构

```
public/BGM/
        │ 构建时 import.meta.glob 扫描
        ▼
lib/bgm-scanner.ts          ← 解析文件路径 → 结构化 Track[]
        │
        ▼
BgmPlayer.tsx               ← 接收 Track[]，UI 完全不变
        │ 点击 📋 按钮
        ▼
BgmPlaylistPopover.tsx      ← 标签栏 + 曲目列表弹出面板
```

### 新增文件

| 文件 | 职责 |
|------|------|
| `src/lib/bgm-scanner.ts` | 构建时扫描 BGM 目录，返回分类后的播放列表 |
| `src/components/ui/BgmPlaylistPopover.tsx` | 播放列表弹出面板 |
| `public/BGM/order.txt` | 情景词顺序模板（初始创建） |

### 修改文件

| 文件 | 改动 |
|------|------|
| `src/components/ui/BgmPlayer.tsx` | 移除硬编码播放列表，接收外部 Track[]；列表按钮接入弹窗 |

## 数据模型

```ts
interface Track {
  category: string;    // 情景词，如 "日常"
  title: string;       // 曲名，如 "悠闲时光"
  audioUrl: string;    // 音频路径，如 "/BGM/日常_悠闲时光.mp3"
  coverUrl?: string;   // 封面路径，如 "/BGM/日常_悠闲时光.jpg"，可选
}

interface BgmScanResult {
  categories: string[];              // 有序情景词列表
  tracks: Record<string, Track[]>;   // 分类 → 曲目列表
  allTracks: Track[];                // 扁平全曲目（用于顺序播放）
}
```

## 播放列表弹窗（BgmPlaylistPopover）

### 触发
- 点击播放器右侧列表按钮 📋 打开
- 点击弹窗外部关闭
- 选择曲目后自动关闭
- 列表按钮在弹窗打开时显示激活态（蓝色高亮）

### 布局

```
┌──────────────────────────┐
│  日常 │ 午后 │ 夜空 │ 夏天  │  ← 情景词标签栏
│──────────────────────────│
│  🖼  悠闲时光      ● 播放中 │  ← 曲目行
│  🖼  清晨薄雾              │
│                          │
└──────────────────────────┘
```

- 标签栏：水平排列，当前分类蓝色边框/文字高亮
- 曲目行：32px 封面缩略图（圆角方形）+ 歌名，当前播放曲蓝色文字
- 无封面时：根据情景词生成纯色渐变占位块

### 视觉风格
- 深色半透明背景（`rgba(20,15,10,0.92)`）+ 毛玻璃模糊
- 米白/奶油色文字体系，延续播放器黑胶复古暖调
- 圆角面板，轻微边框辉光
- 淡入 + 上滑 8px 动画（framer-motion）

### 定位
- 相对于播放器容器绝对定位
- 从播放器右边缘向上方弹出
- 最大高度 320px，超出滚动

## 行为细节

### 无封面回退
按情景词映射纯色渐变：

| 情景词示例 | 渐变色 |
|-----------|--------|
| 日常 | 暖黄 → 奶油 |
| 午后 | 橙黄 → 浅棕 |
| 夜空 | 深蓝紫 → 藏蓝 |
| 夏天 | 翠绿 → 薄荷 |
| 其他 | 灰 → 暖灰（默认） |

具体根据实际 order.txt 中的情景词动态生成 HSL 色相轮换。

### 边界情况
- BGM 目录为空：播放器显示默认状态（无歌曲，"未播放歌曲"）
- order.txt 缺失：所有情景词按拼音排序
- 音频文件无对应封面：使用情景词渐变色块
- 仅有一首歌曲：播放列表仍可打开，显示唯一曲目
- 文件名不含下划线：整个文件名作为 title，category 归入"其他"

## 验收标准
1. 将 mp3 + 封面放入 `public/BGM/`，刷新后播放器自动加载
2. 点击列表按钮，弹出分类标签栏和曲目列表
3. 点击标签切换分类，点击曲目切换播放
4. 播放器原有 UI（唱片、进度条、音量、按钮）保持不变
5. order.txt 控制分类顺序
6. 无封面文件时显示渐变色占位
7. 构建通过，Vercel 部署正常
