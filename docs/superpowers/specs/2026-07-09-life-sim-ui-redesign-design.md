# 人生模拟器 UI 重设计规格文档

> 日期：2026-07-09  
> 主题：LLM Life Sim UI 全面 UI/UX 重设计  
> 状态：待实现

## 1. 设计目标

将当前版本过度使用卡片+留白的布局，改造为「模块分区 + 细线/底色分隔」的紧凑高级界面；同时围绕「动漫日常 + 现代东京」风格，重制上栏、左栏预览面板、正文叙事区、右栏与手机组件，并新增/重制多个全屏大界面。

## 2. 通用规范

- **语言**：全中文化（不影响游戏体验的 logo、视觉装饰除外）。
- **图标**：统一使用 Lucide React 图标；**禁止使用 emoji**。例如好感度使用 `Heart` 图标，而非 ❤ 字符。
- **字体**：沿用项目已有 `font-heading` / `font-body`，保持现代无衬线排版。
- **动画**：使用 Framer Motion 实现悬停、切换、展开的灵动微交互。
- **通知/警告**：前端内部实现提示组件，不调用浏览器原生 `alert` / `confirm`。
- **语义化与可测试性**：使用语义化 HTML5 元素，所有交互元素具备唯一、描述性的 `id`。
- **性能**：避免不必要的重渲染；图标按需使用；动画使用 CSS transform / opacity。

## 3. 整体布局（page.tsx）

### 当前问题

外层 `p-2 gap-2` 加上各区域内部卡片圆角阴影，导致屏幕边缘与卡片之间存在大量无意义空隙。

### 改造方案

- 移除根布局的 `p-2 gap-2`，改为全屏无外边距布局。
- 上栏、左栏、主区域、右栏之间使用 `1px` 柔和分隔线（`border-border-soft`）或背景色微差区分。
- 各区域紧贴视口边缘；内容内部保留最小必要间距。
- 仅对需要浮层感的元素保留圆角/阴影：弹窗、手机、下拉面板、关系网络节点弹窗等。

### 布局骨架

```tsx
<div className="relative z-10 flex flex-col h-screen">
  <TopBar />
  <div className="flex-1 min-h-0 grid grid-cols-1 lg:grid-cols-[auto_1fr_320px]">
    <aside className="hidden lg:flex min-h-0 overflow-hidden">
      <LeftSidebar />
      <SidebarPreviewPanel />
    </aside>
    <main className="min-h-0 overflow-hidden">
      <NarrativePanel />
    </main>
    <aside className="hidden lg:block min-h-0 overflow-hidden">
      <RightPanel />
    </aside>
  </div>
  <Phone />
  <OverlayRenderer />
</div>
```

- 左侧两列合并为一个 `aside`：`LeftSidebar`（按钮列） + `SidebarPreviewPanel`（新增预览面板）。
- 预览面板默认宽度约 `260px`，按钮列约 `56px`；可在断点或用户拖拽下调整（本次实现固定宽度）。

## 4. 上栏（TopBar）

### 4.1 天气球外框

- `TopBar` 设为 `relative`，中间天气球区域使用伪元素或内嵌 `div` 实现「半圆突出式」外框。
- 上栏左右边缘保持水平直线；在天气球正下方切出半圆形容纳区，使 `SkyOrb` 看起来凸出但仍被上栏包裹。
- 天气球本身增加细描边与柔和光晕，强化「被包裹」的视觉效果。
- 保留时间显示在天气球内部或正下方。

### 4.2 左上角日期与节日区

- 平日：左侧显示日历图标 + 日期 + `水曜日（周三）· 黄昏`。
- 节日期间：左侧图标替换为节日对应图片；日期文字下方增加 `今天是{节日名}哦！` 提示，使用纯色圆角矩形底色突出。
- 点击左上角任意区域打开完整日历界面（`calendarFull`）。

### 4.3 右上角 BGM 播放器卡片

- 替换原节日卡片。
- 卡片整体右对齐、顶部对齐，**紧贴视口右上角**，上、右、下与上栏边缘之间不留空隙。
- 结构从左到右：
  1. **唱片图案**：黑胶圆盘 + 唱针，中心黄色圆环为 BGM 封面占位（随歌曲变化）。
  2. **歌曲信息**：封面下方显示 `《歌曲名》`。
  3. **控制按钮**：循环、上一首、播放/暂停、下一首、列表（仅 UI，无真实播放逻辑）。
- 按钮风格参考 `播放器设计.JPG`，颜色适配当前暖橙/日落主题。

## 5. 左栏预览面板（新增 SidebarPreviewPanel）

### 5.1 整体结构

- 左栏由「按钮列 + 预览面板」组成。
- 按钮列收窄（约 `56px`）并左移，为预览面板留出更多空间。
- 预览面板默认显示「个人状态概览」；点击不同按钮切换显示对应预览。
- 每个预览面板底部或头部提供「进入详情」按钮，点击后打开全屏浮层（`OverlayRenderer`）。
- 预览面板文案追求**精简直接、避免换行、对称美学、一眼读取关键信息**。

### 5.2 个人状态预览

- 顶部：ID 卡样式（头像、姓名 `叶悠真`、身份标签 `高中生`、体力/精神/健康三轴进度条）。
- 身体状态概览：
  - 形象化叙述（如 `瘦竹竿一样的颓废年轻人`）。
  - 额外展示生理状态、精神状态标签（如 `营养不良`、`颓废`、`失眠`）。
- 个人讯息概览：年龄、学校/社会身份、详细住址、钱包现金。
- 点击「身体状态」或「个人讯息」卡片进入全屏详情。

### 5.3 天赋才能预览

- 顶部横向展示天赋特质（共感者、夜猫子、过目不忘），使用图标 + 名称。
- 下方展示三大技能树分类：日常、工作、特殊。每个分类只展示最近学习的 1-2 个技能。
- 提供「展开技能树」按钮，进入全屏径向技能树界面。

### 5.4 社交关系预览

- 拆分为两个列表：
  - **在场角色**：当前场景中的角色。
  - **最高好感**：好感度最高的 5 位角色。
- 每个列表最多显示 5 项。
- 列表项：头像（放大）、姓名、关系定位、心形图标 + 好感度数字。
- 底部「关系网络」按钮，进入放射状关系网络大界面。

### 5.5 财富资产预览

- 标题改为「今日流水 + 收支概览」合并展示。
- 显示今日收入、今日支出、钱包现金。
- 银行卡、虚拟财产分开简要列出。
- 底部「财产详情」按钮，进入完整资产大界面。

### 5.6 日历事件预览

- 等比缩小展示「近期日程」（昨日 / 今日 / 明天）。
- 下方展示「世界事件」摘要列表。
- 底部「日历总览」按钮，点击进入网格图表式完整日历界面。

## 6. 正文叙事区（NarrativePanel）

### 6.1 旁白

- 移除气泡包裹，直接呈现在背景上。
- 使用更克制的字体颜色与字号，保持可读性。
- 上下行距压缩约 50%，减少无效空白。

### 6.2 角色对话

- 头像放大，尺寸接近用户红色方框示意。
- 人名字体、对话气泡、说话内容文字同步放大。
- 气泡保留，但上下留白减少。
- 头像与气泡之间的间距适度加大，以平衡放大后的头像。

### 6.3 输入区

- 保持底部输入栏，微调宽度与圆角，与整体紧凑风格一致。

## 7. 右栏（RightPanel）

### 7.1 区域划分（从上到下）

1. **场景印象图**：大幅氛围图，占据右栏顶部较大空间。
2. **GPS 大地图**：迷你地图卡片，带「展开」按钮。
3. **周边动态 + 通知合并区**：合并为一个卡片，分 tab 或上下排列。
4. **手机头部**：改为正常手机顶部边框形态，更宽更扁。

### 7.2 手机改造

- 手机边框改为黑色，屏幕底色改为浅色（不可为黑色），字体重制确保可读。
- 删除右上角「收起」文字，改为 WiFi + 电池图标。
- 收起功能改为点击手机顶部边框或底部外壳边缘。
- 手机整体向右移动，覆盖右侧无用空隙。
- 展开后内容：通知摘要 + 应用网格；进入具体 app 后显示对应界面。

## 8. 全屏大界面重制

### 8.1 技能树大界面

- 采用「中心开花、向四面八方扩散」的径向布局。
- 中心为技能树根节点，分支向四周辐射。
- 节点使用圆形图标 + 名称 + 等级，连线表示前置关系。
- 颜色按日常/工作/特殊分类区分。
- （可选）支持拖拽移动视图、滚轮缩放；本次至少实现静态可缩放视图。

### 8.2 关系网络大界面

- 放射状思维导图：玩家居中，其他角色围绕分布。
- 相同关系定位的角色纳入同一个分组框；唯一关系单独罗列。
- 玩家与角色之间使用实线；角色与角色之间使用虚线。
- 角色节点：空白圆形头像框 + 名称 + 心形图标 + 好感度。
- 右上角小型 UI：
  - 显示/不显示实线
  - 显示/不显示虚线
  - 显示/不显示无实线关系的角色
  - 好感度范围筛选：下限 `【】` ～ 上限 `【】`
- 长按拖动移动视图，滚轮缩放；点击角色头像弹出信息窗口。

### 8.3 日历大界面

- 网格图表式月历，参考 `关于日历的重新设计.JPG`。
- 顶部显示月份、年份；可切换月份。
- 日期格子内标注节日、事件、个人标记。
- 点击日期可查看当日详情。

### 8.4 身体状态 / 个人讯息 / 财富详情 / 系统设置

- 在现有全屏浮层基础上重制内容布局。
- 身体状态：身高、体重、年龄阶段、生理状态、精神状态、疲劳度等。
- 个人讯息：姓名、性别、年龄、生日、国籍、户籍、母语、社会身份、家庭成员、住址、社会评价、证件证书、奖项成就等。
- 财富详情：本周收支、银行卡、虚拟财产、不动产、权益类等。

## 9. 数据类型扩展建议

为支持新界面，建议在 `src/types/index.ts` 中扩展：

- `PlayerBodyState`：增加 `height`、`weight`、`ageStage`、`physiological`、`mental` 等字段。
- `Player`：增加 `age`、`gender`、`birthday`、`nationality`、`householdRegistration`、`nativeLanguage`、`socialIdentity`、`familyMembers`、`address`、`socialEvaluation`、`certificates`、`awards` 等字段。
- `Player.info` 拆分为更完整的个人信息结构。
- `Finance`：明确区分 `cash`、`bankAccounts`、`virtualAssets`、`realEstate`、`equities` 等。
- `CalendarEvent` 与 `DateMark` 已足够支持日历大界面。

## 10. 实现优先级

1. **布局骨架与上栏**：消除卡片留白，重制 TopBar（天气球外框 + BGM 播放器 + 节日迁移）。
2. **左栏预览面板**：新建 `SidebarPreviewPanel`，实现个人状态、天赋才能、社交关系、财富资产、日历事件五个预览。
3. **正文叙事区**：旁白去气泡、对话头像与文字放大。
4. **右栏与手机**：场景印象图、合并周边动态+通知、手机重设计。
5. **大界面**：技能树径向布局、关系网络、日历网格、状态/讯息/财富详情重制。
6. **打磨**：动画、响应式、可访问性、性能优化。

## 11. 验收标准

- 所有参考图片中红色/蓝色/橙色/黄色/绿色/紫色/青色框选区域均按描述改造。
- 无 emoji，全 Lucide 图标。
- 无屏幕边缘与卡片间的无意义留白。
- 所有交互元素具备唯一 `id`。
- 在 1536×738 及以上分辨率下，界面紧凑、美观、信息可读。
- 点击左侧按钮切换预览面板，点击「进入详情」打开全屏浮层。
