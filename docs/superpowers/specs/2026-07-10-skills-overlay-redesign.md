# 技能大界面重做设计

## 背景

当前「天赋才能」区域点击日常/工作/特殊技能面板后进入的大界面过于朴素，缺乏风格化细节。同时存在点击技能图标后技能树导航的问题。需要全面重做，使其匹配游戏日系轻小说/视觉小说的整体美学。

## 目标

1. 全面重做 `TalentsSkillsOverlay`，打造华丽、细节丰富的日系轻小说风格大界面
2. 修复技能图标→技能树的导航问题
3. 统一技能树体验：星座图可视化 + 自定义背景 + 华丽过渡动画

## 设计概览

### 整体架构

将 `TalentsSkillsOverlay` + `SkillTreeOverlay` 合并为单一组件 `SkillsOverlay`，内部维护两种视图模式：

| 模式 | 状态值 | 说明 |
|------|--------|------|
| 总览模式 | `grid` | 三标签页 + 统计横幅 + 技能卡牌阵列 |
| 详情模式 | `tree` | 单个技能的星座树可视化 + 节点详情面板 |

视图切换通过组件内部状态驱动，不再依赖 GameContext 的 `openOverlayView` dispatch，从根本上消除过渡时的竞态问题。

### 标签体系

三个标签并列显示，选中态高亮，无额外"天赋才能"标签前缀：

| 标签 | 图标 | 含义 |
|------|------|------|
| 通用领域 | UtensilsCrossed | 日常生活技能 |
| 专业领域 | Code | 工作/专业技能 |
| 特殊领域 | Wand2 | 特殊/天赋技能 |

## 视觉设计

### 顶部标签栏

```
┌──────────────────────────────────────────┐
│  [🍳 通用领域]   [💻 专业领域]   [✨ 特殊领域]  │
│  ═══════                                  │
└──────────────────────────────────────────┘
```

- 三个标签并列，选中项：文字加粗 + 底部渐变色发光下划线 + 微光晕
- 切换时标签内容区有轻微滑动过渡（左右滑入滑出）
- 每个标签前带对应 Lucide 图标

### 统计横幅

```
┌──────────────────────────────────────────┐
│  ✦ 已习得 2/5 项   可用技能点 ● 3      │
└──────────────────────────────────────────┘
```

- 显示当前分类下的技能掌握进度和可用技能点
- 使用柔和渐变背景色（与标签主色对应）

### 技能卡牌（总览模式）

单张卡牌设计：
```
┌──────────────┐
│  ╭ 装饰边框 ╮ │  ← 解锁/未解锁颜色不同
│  │  🍳      │ │  ← 大图标，带渐变底色
│  │  料理    │ │  ← 技能名称，加粗
│  │  ████░░  │ │  ← 等级进度条
│  │  Lv.3/10│ │  ← 等级文字
│  │  ⭐2    │ │  ← 可用技能点数
│  ╰──────────╯ │
└──────────────┘
```

- 解锁卡牌：白色/奶油色底 + 淡色边框光晕
- 未解锁卡牌：灰色调 + 虚线边框 + 半透明
- 稀有度光效：传说(金)、史诗(紫)、稀有(蓝)、普通(无)
- hover 时浮起动画 + 阴影增强
- 响应式网格：默认4列，窄屏3列，手机2列

### 星座技能树（详情模式）

基于当前 SVG 径向树结构，增强以下方面：

- **画布**：SVG viewBox 100×100，支持用户自定义背景图（通过 `backgroundImage` prop）
- **已解锁节点**：发光圆点 + 实线高亮连线 + 渐变填充
- **未解锁节点**：暗淡圆点 + 虚线连接 + 灰色填充
- **选中节点**：外圈脉冲光环 + 侧边详情面板滑出
- **根节点**（无 parentIds）：比普通节点更大，带双重光环
- **连线**：已解锁连线带渐变色 + 微弱光晕，未解锁连线为虚线

### 节点详情面板

点击树节点后，侧边滑出面板显示：
- 节点名称
- 当前等级 / 最大等级
- 描述文本
- 升级条件（若存在）
- 升级按钮（有足够技能点时可用）

## 过渡动画

### 卡片→技能树展开

点击卡片时：
1. Framer Motion `layoutId` 建立卡片到画布的共享布局关联
2. 卡片原地放大（scale 1→1.5），同级卡片淡出
3. 放大后的卡片边框发光扩散，逐渐演化为全屏技能树画布
4. 技能树节点从卡片中心位置「迸发」弹出，逐个 stagger 到达目标位置
5. 顶部标签栏切换为「← 返回」+ 技能名 + 经验进度条

### 技能树→卡片收起

点击返回时：
1. 节点依次 shrink 回中心
2. 画布收缩回卡片尺寸
3. 周围卡片淡入
4. 恢复总览模式布局

### 标签切换

- 内容区水平滑动过渡（方向取决于切换方向）
- 标签下划线平滑移动到新标签位置

## 成功标准

- 点击侧边栏技能分类卡片，以华丽动画打开重做后的 SkillsOverlay
- 三个标签（通用/专业/特殊领域）可流畅切换，带滑动过渡
- 技能卡牌展示精美，含进度条、稀有度光效、hover 动效
- 点击技能卡牌，以卡片放大展开动画过渡到星座技能树
- 技能树节点可点击查看详情面板
- 返回按钮从技能树回到总览，收起动画流畅
- 自定义背景图可配置
- TypeScript 类型检查通过
- 不影响页面其他交互

## 依赖与风险

- 依赖：framer-motion（已有）、lucide-react（已有）、Tailwind CSS（已有）
- 风险：`layoutId` 共享布局动画在复杂嵌套场景下可能有性能问题，需控制节点 stagger 数量
- 风险：删除旧组件需确认无其他未发现的引用
