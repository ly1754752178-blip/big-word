# Task 7: 自检验证

## 目标
在真实运行环境中验证个人状态界面和通知系统，确保功能、UI、交互、稳定性均符合要求。

## 验证步骤
1. 启动开发服务器：`npm run dev`。
2. 打开浏览器访问应用（通常为 http://localhost:5173 或控制台显示的地址）。
3. 在浏览器中执行以下验证：
   - 左侧 `StatusPreview` 显示头像、姓名、身份标签、三条状态条、当前状态、身体摘要。
   - 点击 `StatusPreview` 能打开全屏 `PersonalStatusOverlay`。
   - `PersonalStatusOverlay` 中无「疲劳」相关内容。
   - 身份锚点区显示头像、姓名、身份、年龄、三条状态条。
   - 身体状态区显示身高/体重/年龄阶段、生理/精神状态标签、描述。
   - 个人讯息区显示完整档案字段。
   - 资质荣誉区显示证书/奖项。
   - 详情页加载后右上角出现 info 通知。
   - 通知 hover 暂停，关闭按钮可关闭，4 秒后自动消失。
   - 在不同分辨率下（375px / 768px / 1440px）布局正常。
4. 检查浏览器控制台无报错。

## 需要记录的结果
- 截图（可选但推荐）：桌面端详情页、移动端详情页、通知显示。
- 验证项通过/失败清单。
- 发现的任何问题。

## 验证完成后
1. 停止开发服务器。
2. 将验证报告写入 `.superpowers/sdd/reports/task-7-report.md`。
3. 更新 `.superpowers/sdd/progress.md` 中 Task 7 为完成状态。
4. 提交最终报告（可选）：
```
docs: 个人状态重制验证报告
```

## 全局约束
- 禁用 emoji，验证报告中也不例外。
- 记录真实观察结果，不虚构。
