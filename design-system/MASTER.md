# 综漫日本生活模拟器 — 设计系统（Design System）

> 美学方向：蔚蓝档案（Blue Archive）的清新明快 + 日常系 Galgame 的柔和诗意。
> 底层约束：现实日本现代社会背景，全中文界面，禁用 emoji，使用 Lucide 图标。

---

## 1. 设计关键词

- **清新（Fresh）**：如清晨学园屋顶的空气，明亮但不刺眼。
- **柔和（Soft）**：低饱和、高明度，像 Galgame 中黄昏教室的光线。
- **真实日常（Everyday）**：便利店、电车、公寓、咖啡馆，不是未来都市。
- **灵动（Alive）**：微交互动画让界面有呼吸感，鼓励点击与探索。
- **高级（Premium）**：拒绝廉价扁平，讲究层次、质感、细节。

## 2. 色彩系统

### 主色板

| Token | Hex | 用途 |
|---|---|---|
| `--sky-50` | `#F0F9FF` | 页面背景、卡片底色 |
| `--sky-100` | `#E0F2FE` |  hover 背景、轻量强调 |
| `--sky-300` | `#7DD3FC` | 主按钮、激活态、进度条 |
| `--sky-500` | `#0EA5E9` | 链接、高亮、关键图标 |
| `--coral-50` | `#FFF1F2` | 情感/关系模块背景 |
| `--coral-300` | `#FDA4AF` | 好感度、温馨提示 |
| `--coral-500` | `#F43F5E` | 重要通知、紧急状态 |
| `--mint-50` | `#F0FDF4` | 健康/体力模块背景 |
| `--mint-300` | `#86EFAC` | 状态恢复、正向反馈 |
| `--mint-500` | `#22C55E` | 成功、完成 |
| `--cream-50` | `#FFFBEB` | 叙事区背景、便签感 |
| `--cream-100` | `#FEF3C7` | 选中标签、温柔高亮 |
| `--lavender-50` | `#F5F3FF` | 天赋/创作模块背景 |
| `--lavender-300` | `#C4B5FD` | 技能、创意、艺术相关 |
| `--slate-800` | `#1E293B` | 主文字 |
| `--slate-500` | `#64748B` | 次要文字 |
| `--slate-300` | `#CBD5E1` | 边框、分隔线 |
| `--white` | `#FFFFFF` | 卡片表面、浮层 |

### 色彩原则

- 背景使用极浅的低饱和色，避免纯白刺眼。
- 功能模块按语义色分区（关系=珊瑚、健康=薄荷、创作=薰衣草、系统=天空蓝）。
- 文字对比度 ≥ 4.5:1，确保可读性。
- 不使用高饱和霓虹色，保持 Galgame 的日常温柔感。

## 3. 字体系统

### 字体栈

```css
--font-display: 'Noto Sans JP', 'M PLUS Rounded 1c', 'PingFang SC', 'Microsoft YaHei', sans-serif;
--font-body: 'Noto Sans JP', 'PingFang SC', 'Microsoft YaHei', sans-serif;
--font-mono: 'JetBrains Mono', 'SF Mono', monospace;
--font-number: 'Outfit', 'DIN Alternate', 'Helvetica Neue', sans-serif;
```

### 字号规范

| 层级 | 大小 | 字重 | 用途 |
|---|---|---|---|
| Display | 32-40px | 700 | 页面大标题、角色名 |
| H1 | 24px | 700 | 模块标题 |
| H2 | 18px | 600 | 卡片标题 |
| H3 | 15px | 600 | 小标题 |
| Body | 14px | 400 | 正文 |
| Caption | 12px | 400 | 辅助说明、时间戳 |
| Number | 14-18px | 500 | 数据、金额、体力值 |

### 字体原则

- 使用现代无衬线字体，中文显示清晰。
- 数字使用 Outfit / DIN Alternate 等几何感字体，增强数据可读性。
- 行高 1.5-1.75，段落舒适。
- 重要文字适度加粗，避免全篇细体。

## 4. 间距与布局

### 间距比例

基于 4px 网格：4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 96。

### 布局原则

- **桌面端**：三栏布局（左导航/预览 240-280px，中间叙事区 1fr，右信息/地图 320px）。
- **平板端**：折叠为两栏或抽屉。
- **移动端**：底部导航 + 全屏模态框。
- 卡片间距 12-16px，模块间距 24-32px。
- 内容区最大宽度不过度拉伸，保持阅读舒适。

## 5. 组件风格

### 卡片（Card）

- 圆角：`16-20px`
- 背景：白色或极浅奶油色，带微妙渐变
- 阴影：`0 2px 12px rgba(30, 41, 59, 0.06)`
- hover：轻微上浮 + 阴影加深 + 边框色变化
- 不使用生硬的直角或纯扁平

### 按钮（Button）

- 主按钮：天空蓝渐变背景，白色文字，圆角 12px
- 次按钮：白色背景 + 浅色边框 + 图标
- Ghost 按钮：仅文字/图标，hover 背景变化
- 按压反馈：scale 0.97，150ms ease-out
- 图标按钮：44×44px 最小点击区域

### 输入框（Input）

- 圆角 12px，浅灰边框
- focus：天空蓝边框 + 柔和光晕
- placeholder 不使用作标签
- 错误提示出现在字段下方

### 模态框/浮层（Modal/Overlay）

- 从触发点或屏幕中心缩放进入
- 背景 scrim：40-60% 黑色半透明 + 轻微 backdrop-blur
- 顶部强调条：按模块语义色渐变
- 关闭按钮始终可见

### 标签/徽章（Badge）

- 圆角 pill（9999px）
- 轻量背景色 + 对应深色文字
- 用于状态、地点、关系标签

## 6. 动效系统

### 时间规范

- 微交互：150-200ms
- 模态进入/退出：250-300ms
- 复杂切换：400ms 以内
- 使用 ease-out 进入，ease-in 退出

### 动画类型

- **Hover**：卡片上浮 2-4px，阴影加深，图标轻微放大。
- **Press**：按钮 scale 0.97，150ms。
- **Enter**：元素从下方 12px 淡入，stagger 30-50ms。
- **Modal**：从 0.96 scale + 透明度 0 到正常，spring 曲线。
- **Tab 切换**：内容淡入淡出，指示器滑动。
- **Loading**：骨架屏或柔和脉冲，不使用生硬 spinner。

### 性能原则

- 仅使用 transform / opacity 动画。
- 尊重 `prefers-reduced-motion`。
- 动画不阻塞交互。

---

## 7. 图标规范

- 统一使用 **Lucide React**。
- 线框风格（outline），stroke-width 1.5-2。
- 同一层级不混用填充与线框图标。
- 图标尺寸规范：sm 16px，md 20px，lg 24px，xl 32px。
- 图标按钮必须带 `aria-label`。

## 8. 意象与氛围

### Blue Archive 可取之处（保留）

- 圆角卡片与柔和阴影
- 清新的蓝白基调
- 角色头像圆形或圆角矩形
- 信息层级清晰、 youthful 的排版
- 轻量渐变与光晕

### Blue Archive 需弱化之处

- 未来科技感（全息、金属、锐角）
- 过高饱和的战斗元素
- 复杂机能风 UI

### Galgame 日常氛围注入

- 大量使用奶油色、柔粉、薄荷等暖调
- 叙事区模拟纸面/便签质感
- 时间、季节、天气作为情感化元素
- 对话文字有节奏地出现（可选打字机效果）
- 角色互动以温柔、细腻为主

## 9. 响应式断点

| 断点 | 宽度 | 布局 |
|---|---|---|
| Mobile | < 640px | 单栏，底部导航，抽屉式侧边栏 |
| Tablet | 640-1024px | 两栏，侧边栏可折叠 |
| Desktop | > 1024px | 三栏固定布局 |

---

## 10. 设计原则总结

1. **不做最小可行产品**——每个组件都追求细节与质感。
2. **以现实生活为骨，动漫角色为肉**——UI 服务于真实日本日常模拟。
3. **动效有意义**——每个动画都回应用户操作，而非装饰。
4. **信息有呼吸**——留白充足，层级清晰，不拥挤。
5. **全中文、无 emoji、高级图标**——保持统一语言。
